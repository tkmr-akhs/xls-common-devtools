# ADR 0008: Allow Runtime Temporary Module Cleanup by Naming Pattern in Lib_UnitTest

- Status: Accepted
- Date: 2026-05-23

## Context

`Lib_UnitTest`'s `pRemoveRuntimeRunnerModule` deletes not only the temporary module created by the current run, but also any standard module whose name matches `Tmp_UTRUN[A-Z]{22}`.

This means a user-created or investigation module that accidentally matches the same naming pattern could be deleted.

## Decision

Treat standard modules matching `Tmp_UTRUN[A-Z]{22}` as test-runner temporary modules and keep deleting them.

Because the probability is low and there is little reason to use such a name for ordinary user modules, we will not change the implementation to track and delete only modules generated by the current run.

## Consequences

- Temporary-module cleanup during `UnitTestMain` keeps the current approach.
- Developers must not manually create standard modules whose names consist of `Tmp_UTRUN` plus 22 uppercase letters.
- This decision records operational safety handling, not test result judgment.
